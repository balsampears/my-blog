**同源策略**  
浏览器出于安全考虑，限制当前域名只能访问同一**协议、域名、端口**的网站资源。当请求另一个网站违反了同源策略时，就叫“跨域”，会被浏览器拦截。


## 解决方案
存在a.com前端，连接a.com/api后端；b.com前端，连接b.com/api后端。  
当a.com请求b.com和b.com/api时，触发跨域。

### 1. 被调用方添加跨域许可（CORS）
CORS是跨域资源共享，配置之后可以让origin源访问当前服务。  

Gin框架中，在b.com/api后端添加以下代码：
```
c.Header("Access-Control-Allow-Origin", "https://a.com")           // 必须在白名单内
c.Header("Access-Control-Allow-Credentials", "true")
c.Header("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS")
c.Header("Access-Control-Allow-Headers", "Authorization, Content-Type, Accept")
c.Header("Access-Control-Max-Age", "86400")
```

### 2. 调用方添加nginx反向代理
在调用方反向代理来隐藏自身，让目标以为来源是目标同源地址。

在a.com前端所在的服务器上的nginx添加以下配置；
```
    # 代理到 b.com，对外路径是 /b-api/
    location /b-api/ {
        proxy_pass https://b.com/api/;

        proxy_set_header Host b.com;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        # 若 b 需要知道真实来源（可选）
        proxy_set_header X-Forwarded-Host $host;
    }
```
